Getting Started

Set up your machine for the From Vibe Coding to Agentic Engineering workshop. Follow the steps below in order — should take ~15 minutes.

Your OS
  1. VS Code & GitHub Copilot
  2. Shell setup
  3. GitHub CLI
  4. Sign in with your EMU account
  5. GitHub Copilot CLI
  6. Clone the workshop repo
  7. Run the backend
  8. Run the frontend
  9. Seed sample data
  10. Verify everything works
1

VS Code & GitHub Copilot

Install the latest Visual Studio Code.

On a Microsoft-managed device, GitHub Copilot Chat is typically pre-installed and signed in via your corporate account. Open the Copilot panel (the chat icon in the Activity Bar) to confirm.

If Copilot is not present, install these extensions from the Marketplace:

  • GitHub Copilot
  • GitHub Copilot Chat

Or run from a terminal:

code --install-extension GitHub.copilot
code --install-extension GitHub.copilot-chat
2

Shell setup PowerShell 7+ zsh / Homebrew bash / package manager

The workshop assumes PowerShell 7 or newer (not the old Windows PowerShell 5.1 that ships with Windows). Check your version:

$PSVersionTable.PSVersion

If the Major version is below 7, install it via winget:

winget install --id Microsoft.PowerShell --source winget
Tip: Set PowerShell 7 (pwsh) as the default terminal in VS Code: open the Command Palette (Ctrl+Shift+P) → Terminal: Select Default Profile → PowerShell.

macOS already ships with a modern shell (zsh). The only thing you need is Homebrew — we'll use it to install everything else.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

After it finishes, follow the on-screen instructions to add brew to your PATH, then verify:

brew --version

Most Linux distros ship with bash and a package manager — nothing to install. Just make sure curl and git are available:

# Debian / Ubuntu
sudo apt update && sudo apt install -y curl git

# Fedora / RHEL
sudo dnf install -y curl git

# Arch
sudo pacman -S --needed curl git
3

GitHub CLI

Install the GitHub CLI:

winget install --id GitHub.cli --source winget
brew install gh
# Debian / Ubuntu
(type -p wget >/dev/null || (sudo apt update && sudo apt install wget -y)) \
  && sudo mkdir -p -m 755 /etc/apt/keyrings \
  && wget -qO- https://cli.github.com/packages/githubcli-archive-keyring.gpg \
     | sudo tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null \
  && sudo chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
  && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" \
     | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
  && sudo apt update \
  && sudo apt install gh -y

# Fedora / RHEL
sudo dnf install gh

# Arch
sudo pacman -S github-cli

Full install matrix: cli/cli install_linux.md.

Close and reopen your terminal, then verify:

gh --version
4

Sign in with your EMU account

Microsoft employees use an Enterprise Managed User (EMU) GitHub account — typically your alias_microsoft username. Authenticate the CLI with it: 

gh auth login

Answer the interactive prompts as follows:

  • Where do you use GitHub? → GitHub.com
  • Preferred protocol → HTTPS
  • Authenticate Git with your GitHub credentials? → Yes
  • How would you like to authenticate? → Login with a web browser

A browser window will open. Sign in with your Microsoft alias (the EMU account you use for github.com/microsoft), not your personal GitHub.

Confirm you are logged in as the right user:

gh auth status
Already logged in as a personal account? Add the EMU account alongside it: 
gh auth login --hostname github.com
Then switch with gh auth switch.
5

GitHub Copilot CLI

Install the standalone GitHub Copilot CLI — an agentic coding assistant that runs directly in your terminal. We'll use it during the workshop to run Ralph loops (long-running, autonomous agent loops that iterate on a task without manual prompting). 

winget install --id GitHub.Copilot --source winget

Copilot CLI is distributed as an npm package and requires Node.js 22+. Install Node via Homebrew first if you don't have it:

brew install node
npm install -g @github/copilot

Copilot CLI is distributed as an npm package and requires Node.js 22+. Install Node first if you don't have it:

# Install Node 22 (NodeSource — Debian / Ubuntu / RHEL)
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Then install Copilot CLI
sudo npm install -g @github/copilot

Verify and sign in:

copilot --version
copilot

On first launch, Copilot CLI will prompt you to authenticate. Use the same EMU / Microsoft alias account you signed in with for gh.

6

Clone the workshop repo

Pick a folder you like and clone with the CLI (it uses your gh credentials, no extra setup needed):

gh repo clone tezansahu/microsoft-idc-interns-agentic-engineering-workshop-2026
cd microsoft-idc-interns-agentic-engineering-workshop-2026

Open it in VS Code:

code .
7

Run the backend Python 3.10+

Make sure you have Python 3.10+ installed.

Check with python --version. Then, from the repo root:

cd backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m uvicorn app.main:app --port 8000
Activation blocked? If PowerShell refuses to run Activate.ps1, allow local scripts for the current user once: 
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

macOS ships with an older Python — install a modern one via Homebrew if needed:

brew install python@3.12

Then, from the repo root:

cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m uvicorn app.main:app --port 8000

Most distros include Python 3.10+. Install the venv module if you don't have it:

# Debian / Ubuntu
sudo apt install -y python3 python3-venv python3-pip

# Fedora / RHEL
sudo dnf install -y python3 python3-pip

Then, from the repo root:

cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m uvicorn app.main:app --port 8000

API runs at http://localhost:8000 · Interactive docs at /docs.

8

Run the frontend Node.js 20+

You'll need Node.js 20 or newer (node --version to check). If you installed Copilot CLI in step 5 you already have Node 22+.

If Node is missing:

winget install --id OpenJS.NodeJS.LTS --source winget

If Node is missing:

brew install node

If Node is missing, install Node 22 from NodeSource (see step 5).

Then, in a new terminal (leave the backend running):

cd frontend
npm install
npm run dev

The app runs at http://localhost:5173.

9

Seed sample data

The dashboard charts only look meaningful once there is data. Populate the store with a few weeks of sample entries:

cd backend
.venv\Scripts\Activate.ps1
python seed.py
cd backend
source .venv/bin/activate
python seed.py
cd backend
source .venv/bin/activate
python seed.py

Refresh the frontend — you should see the timeline chart and calendar heatmap come alive.

10

Verify everything works

  • ✅ code --version works and Copilot Chat opens in VS Code
  • ✅ $PSVersionTable.PSVersion shows Major ≥ 7
  • ✅ brew --version prints a version
  • ✅ git --version and curl --version both work
  • ✅ gh auth status shows your EMU account on github.com
  • ✅ copilot --version prints a version and you can launch copilot
  • ✅ Backend responds at localhost:8000/docs
  • ✅ Frontend loads at localhost:5173 with charts populated
You're ready. Head back to the repo and open briefs/ to see what you'll be building during the workshop.
